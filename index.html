レンタルサーバー
開発セットアップご案内

このページでは、レンタルサーバーをご契約いただいた方へ、
開発を始めるために必要な 3つの設定 を、
画面付き・手順どおりにご案内いたします。

パソコンの操作に不慣れな方でも安心して進められるよう、
一つひとつ丁寧に説明しております。
目次（クリックで移動）
  1. サーバーへの接続情報の準備（SSH鍵の設定）
  2. 専用データベースの追加（MariaDB）
  3. メールの設定（SMTPサーバー）

1. サーバーへの接続情報の準備（SSH鍵の設定）

開発者がサーバーに安全にアクセスするために、
「鍵」を使った接続方法を設定します。

SSH鍵とは、サーバーに安全に接続するための「デジタルな鍵」です。
家の鍵と同じように、正しい鍵を持っている人だけがサーバーに入れます。
公開鍵（鍵穴）をサーバーに置き、秘密鍵（鍵本体）を開発者に渡します。
┌───────────────────────────────────────── │ SSH鍵のイメージ │ │ 【公開鍵】＝ 鍵穴 【秘密鍵】＝ 鍵 │ サーバーに置く 開発者が持つ │ │ ┌─────────┐ 一致すると ┌─────────┐ │ │ 鍵穴 │ ◄─────────► │ 鍵 │ │ │ サーバー │ 接続OK! │ 開発者 │ │ └─────────┘ └─────────┘ └─────────────────────────────────────────

全体の流れ

STEP 1 STEP 2 STEP 3 STEP 4 ┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐ │ SSHを │──►│ 鍵を │──►│ 公開鍵 │──►│ 秘密鍵 │ │ ONにする│ │ 作成 │ │ を確認 │ │ を送る │ └────────┘ └────────┘ └────────┘ └────────┘
1 サーバーパネルにログインして、SSHをONにする

まずはサーバーの管理画面にログインし、SSH機能を有効にします。

1-1. サーバーパネルにログイン

お使いのブラウザ（Chrome、Edge、Safariなど）を開いてください。
画面の一番上にある「アドレスバー」に、以下の文字を入力して Enter キーを押します。

ブラウザのアドレスバーに以下を入力して Enter: https://secure.xserver.ne.jp/xapanel/login/ アドレスバーとは: ┌──────────────────────────────────────────────┐ │ ← → O [ https://secure.xserver.ne.jp/... ] │ │ ↑ここに入力します │ └──────────────────────────────────────────────┘ ↓ ログイン画面が表示されます ┌────────────────────────────────── │ サーバーパネル ログイン画面 │ │ サーバーID: [____________] │ パスワード: [____________] │ │ [ ログイン ] └────────────────────────────────── ※ サーバーIDとパスワードは、契約時に届いた 「サーバーアカウント設定完了のお知らせ」 というメールに書かれています。 ※ メールが見つからない場合は、受信トレイで 「Xserver」「設定完了」で検索してみてください。
ログインに成功すると、サーバーパネルのトップ画面が表示されます。
たくさんのメニュー項目が並んだ管理画面です。

1-2. SSH設定を開く

サーバーパネルの画面から「SSH設定」を探してクリックします。

サーバーパネルのメニューから探します: ┌─────────────────────────────── │ サーバーパネル メニュー │ │ ▶ アカウント │ ▶ ホームページ │ ▶ メール │ ▶ FTP │ ▶ データベース │ ▶ SSH設定 ◄── ここをクリック! │ ▶ セキュリティ └─────────────────────────────── ※ 見つからない場合は、画面を下にスクロール (マウスのホイールを手前に回す)してください。

1-3. SSH設定をONにする

SSH設定の画面が開いたら、まずSSH機能を「ON」に切り替えます。
初めは「OFF」になっているので、「ON」にする必要があります。

┌────────────────────────────────────── │ SSH設定 画面 │ │ 現在の設定: OFF │ │ SSH設定を変更する: │ │ ○ OFF(無効) │ ● ON(有効) ◄── ONを選んでください! │ │ [ 設定する ] ◄── クリック! └──────────────────────────────────────
「設定する」をクリックすると、「SSH設定の変更が完了しました」
というメッセージが表示されます。これでSSHが使えるようになりました。
2 鍵ペア（公開鍵と秘密鍵）を作成する

SSHをONにしたら、次は「鍵」を作ります。
サーバーが自動で「公開鍵」と「秘密鍵」のセットを作ってくれます。

2-1. 公開鍵認証用鍵ペアの生成タブを開く

┌────────────────────────────────────────────── │ SSH設定 画面 │ │ [公開鍵認証用鍵ペアの生成] [公開鍵の登録・更新] │ ↑ │ このタブをクリック! │ │ パスフレーズ: [____________] │ パスフレーズ(確認): [____________] │ │ ※ パスフレーズとは「鍵のパスワード」のことです。 │ ※ お好きなパスワードを入力してください。 │ ※ 2つの欄に同じものを入力します。 │ │ [ 確認画面へ進む ] └────────────────────────────────────────────── パスフレーズの例: MyKey2024!pass (英字の大小・数字・記号を混ぜると安全です)
パスフレーズは絶対に忘れないよう、紙にメモして保管してください。
開発者にも後でお伝えする必要があります。

2-2. 確認して生成する

┌────────────────────────────────────── │ 確認画面 │ │ 以下の内容で鍵ペアを生成しますか? │ │ [ 生成する ] ◄── クリック! └────────────────────────────────────── ↓ すぐに秘密鍵ファイルのダウンロードが始まります!
「生成する」を押すと、秘密鍵のファイルが自動でダウンロードされます。
ファイル名は サーバーID.key のような名前です。
ダウンロードされたファイル: パソコンの「ダウンロード」フォルダを開いてください。 ┌────────────────────────────────────── │ ダウンロード フォルダ │ │ xs123456.key ◄── これが秘密鍵! │ │ ※ このファイルはとても大切です。 │ 開発者に渡すまで、なくさないように │ 保管してください。 └────────────────────────────────────── ダウンロードフォルダの開き方: ・Windows: エクスプローラー → ダウンロード ・Mac: Finder → ダウンロード
3 公開鍵がサーバーに登録されたことを確認する

鍵ペアの生成が完了すると、公開鍵は自動的にサーバーに登録されます。
念のため、ちゃんと登録されたか確認しましょう。

SSH設定画面で「公開鍵の登録・更新」タブを開きます: ┌────────────────────────────────────────────── │ SSH設定 画面 │ │ [公開鍵認証用鍵ペアの生成] [公開鍵の登録・更新] │ ↑ │ このタブで確認! │ │ 登録済み公開鍵: │ ┌──────────────────────────────────────┐ │ │ ssh-rsa AAAAB3NzaC1yc2EAAA.... │ │ │ (とても長い英数字の並び) │ │ └──────────────────────────────────────┘ │ │ ↑ ここに文字列が表示されていれば成功です! └──────────────────────────────────────────────
「登録済み公開鍵」の欄に、長い文字列が表示されていれば成功です。
公開鍵（鍵穴）はサーバー側に設置完了しています。
特に何も操作する必要はありません。確認だけでOKです。
4 秘密鍵ファイルとサーバー情報を開発者に送る

ダウンロードした秘密鍵ファイルと、サーバーの接続情報を
安全な方法で開発者にお送りください。

4-1. サーバー情報を確認する

サーバーパネルの左メニューから「サーバー情報」をクリックして、
ホスト名を確認します。

┌────────────────────────────────────── │ サーバー情報 画面 │ │ ホスト名: sv12345.xserver.jp │ サーバーID: xs123456 │ ... └────────────────────────────────────── ※ ホスト名は「サーバーID.xsrv.jp」の形式でも 接続できます。 例: xs123456.xsrv.jp

4-2. 開発者に送る情報

送る情報は以下の3つです: ┌────────────────────────────────────────── │ │ 【A】秘密鍵ファイル │ ( xs123456.key ) │ │ 【B】パスフレーズ │ (鍵を作ったときに設定したパスワード) │ │ 【C】サーバー接続情報: │ ・ホスト名: xs123456.xsrv.jp │ ・ユーザー名: xs123456 │ ・ポート番号: 10022 │ └──────────────────────────────────────────
秘密鍵はサーバーの「鍵」そのものです。
安全に送るために、情報を2回に分けて送ることをおすすめします：

1回目（メール）：秘密鍵ファイルをパスワード付きZIPで添付 + サーバー情報
2回目（チャットや電話など別の方法）：パスフレーズとZIPのパスワード

こうすることで、万が一メールが他人に見られても安全です。
パスワード付きZIPの作り方（Windows）：
ファイルを右クリック →「送る」→「圧縮(zip形式)フォルダー」
※ Windows標準ではパスワード設定できないため、
　 7-Zip などの無料ソフトを使うと便利です。

パスワード付きZIPの作り方（Mac）：
ターミナルで zip -e ファイル名.zip 元ファイル
開発者が3つの情報（鍵ファイル・パスフレーズ・サーバー情報）を
受け取ったら、サーバーへの接続テストを行います。
開発者から「接続できました」と連絡があれば完了です！

2. 専用データベースの追加（MariaDB）

開発者が使うための専用のデータベース（データの保管庫）を作成します。
3つのことを行います：データベースの作成、ユーザーの作成、権限の設定です。

データベースとは、Webサイトのデータ（記事、会員情報など）を
整理して保管しておく「倉庫」のようなものです。
データベースのイメージ: ┌──────────┐ │ Webサイト │ └────┬─────┘ │ データを読み書き ▼ ┌────────────────────────────┐ │ データベース (MariaDB) │ │ ┌──────┐┌──────┐┌──────┐ │ │ │会員表││記事表││注文表│ │ │ └──────┘└──────┘└──────┘ │ └────────────────────────────┘

全体の流れ

STEP 1 STEP 2 STEP 3 ┌──────────┐ ┌──────────┐ ┌──────────┐ │DB を作成 │──►│ユーザーを│──►│ 権限設定 │ │ │ │ 作成 │ │+情報を送る│ └──────────┘ └──────────┘ └──────────┘
1 データベースを作成する

1-1. MySQL設定を開く

サーバーパネルのメニューから探します: ┌─────────────────────────────────────── │ サーバーパネル メニュー │ │ ▶ アカウント │ ▶ ホームページ │ ▶ メール │ ▶ FTP │ ▶ データベース │ └─ MySQL設定 ◄── ここをクリック! │ ▶ SSH設定 └───────────────────────────────────────

1-2. 新しいデータベースを追加する

┌────────────────────────────────────────────── │ MySQL設定 画面 │ │ [MySQL一覧] [MySQL追加] [MySQLユーザ一覧]... │ ↑ │ このタブをクリック! │ │ データベース名: │ ┌────────────┬────────────────┐ │ │ xs123456_ │ dev_project │ │ └────────────┴────────────────┘ │ ↑自動で入る ↑ここに名前を入力 │ │ 文字コード: UTF-8 (そのままでOK) │ │ [ 確認画面へ進む ] └────────────────────────────────────────────── 入力例: dev_project → 実際の名前は「xs123456_dev_project」になります
確認画面で「追加する」をクリックすると、
データベースが作成されます。
xs123456_dev_project のような名前で一覧に表示されます。
2 データベース用のユーザーを作成する
┌────────────────────────────────────────────── │ MySQL設定 画面 │ │ [MySQL一覧] [MySQL追加] [MySQLユーザ追加]... │ ↑ │ このタブをクリック! │ │ ユーザーID: │ ┌────────────┬────────────────┐ │ │ xs123456_ │ devuser │ │ └────────────┴────────────────┘ │ ↑自動で入る ↑ここに名前を入力 │ │ パスワード: [ 強力なパスワードを入力 ] │ 確認: [ 同じパスワードをもう一度 ] │ │ [ 確認画面へ進む ] └────────────────────────────────────────────── 入力例: ユーザーID: devuser パスワード: aBcD1234!xYz (英数字・記号を混ぜた強いパスワード)
パスワードは推測されにくいものにしてください。
英大文字・小文字・数字・記号を混ぜると安全です。
このパスワードはメモしておいてください。
「追加する」をクリックすると、
ユーザー xs123456_devuser が作成されます。
3 データベースにユーザーの権限を設定する
┌────────────────────────────────────────────── │ MySQL設定 画面 → [MySQL一覧] タブ │ │ データベース名 アクセス権設定 │ ────────────────────────────────── │ xs123456_dev_project アクセス権未所有ユーザ │ ┌────────────────────┐ │ │ xs123456_devuser ▼ │ │ └────────────────────┘ │ [ 追加 ] ◄── クリック! └──────────────────────────────────────────────
「追加」をクリックすると、
ユーザーがデータベースにアクセスできるようになります。
「アクセス権所有ユーザ」の欄に名前が表示されれば成功です。

開発者に送る情報

以下の情報を開発者にお送りください: ┌────────────────────────────────────────────── │ │ 1. ホスト名: localhost │ (または mysql○○.xserver.jp ※パネルに記載) │ │ 2. データベース名: xs123456_dev_project │ │ 3. ユーザー名: xs123456_devuser │ │ 4. パスワード: (設定したパスワード) │ │ 5. ポート番号: 3306 │ └──────────────────────────────────────────────
ホスト名は、サーバーパネルの
「MySQL設定」→「MySQL一覧」の画面下部に記載されています。
mysql○○.xserver.jp のような形式です。
パスワードは鍵ファイルとは別の方法で送りましょう。
例：データベース名をメールで送り、パスワードはチャットで送る。

3. メールの設定（SMTPサーバー）

Webサイトからメール（お問い合わせ通知、パスワードリセットなど）を
送信するための設定情報を、開発者にお伝えします。

SMTPサーバーとは、メールを「送る」ための仕組みです。
郵便局のようなもので、Webサイトが書いた手紙を届けてくれます。
メール送信のイメージ: ┌──────────┐ ┌──────────┐ ┌──────────┐ │ Webサイト │──► │ SMTP │──► │ 受信者 │ │(手紙を書く)│ │ サーバー │ │(届く) │ └──────────┘ │ (郵便局) │ └──────────┘ └──────────┘

全体の流れ

STEP 1 STEP 2 STEP 3 ┌──────────┐ ┌──────────┐ ┌──────────┐ │ メール │──►│ SMTP情報 │──►│ 情報を │ │アカウント│ │ を確認 │ │ 開発者に │ │ を作成 │ │ │ │ 送る │ └──────────┘ └──────────┘ └──────────┘
1 メールアカウントを作成する

1-1. メールアカウント設定を開く

サーバーパネルのメニューから探します: ┌─────────────────────────────────────────── │ サーバーパネル メニュー │ │ ▶ アカウント │ ▶ ホームページ │ ▶ メール │ └─ メールアカウント設定 ◄── ここ! │ ▶ FTP │ ▶ データベース └─────────────────────────────────────────── → ドメイン選択画面が出たら、 使いたいドメインの「選択する」をクリック

1-2. 新しいメールアカウントを追加する

┌────────────────────────────────────────────── │ メールアカウント設定 画面 │ │ [メールアカウント一覧] [メールアカウント追加] │ ↑ │ このタブをクリック! │ │ メールアカウント: │ ┌──────────────┬──┬──────────────┐ │ │ noreply │@ │ example.com │ │ └──────────────┘ └──────────────┘ │ ↑好きな名前 ↑ドメインを選択 │ │ パスワード: [ 強力なパスワードを入力 ] │ 確認: [ 同じパスワードをもう一度 ] │ │ 容量: 2000 MB (そのままでOK) │ │ [ 確認画面へ進む ] └────────────────────────────────────────────── 入力例: メールアカウント: noreply @ example.com → noreply@example.com ができます パスワード: Str0ng!Pass#2024
用途に合わせてアドレスを決めましょう。
例：noreply@（自動送信用）、info@（問い合わせ用）
「追加する」をクリックすると、
メールアカウントが作成されます。
一覧に新しいアカウントが表示されます。
2 SMTPサーバー情報を確認する

メール送信に必要なSMTP情報は、サーバーパネルで確認できます。

サーバーパネル → メール → メールアカウント設定 → 対象ドメインを選択 ┌────────────────────────────────────────── │ メールソフト設定 情報 │ │ ■ 送信メール(SMTP)サーバー │ │ サーバー名: sv○○○○.xserver.jp │ ポート番号: 465 │ 暗号化: SSL │ 認証: あり │ │ ■ 受信メール(POP/IMAP)サーバー │ │ サーバー名: sv○○○○.xserver.jp │ ポート番号: 993 (IMAP) / 995 (POP) │ 暗号化: SSL └────────────────────────────────────────── ※ sv○○○○ の部分は、お客様のサーバーによって 異なります。
上記の情報が画面に表示されます。
この画面をスクリーンショット（画面の写真）で保存しておくと便利です。
スクリーンショットの撮り方：
- Windows: Windowsキー + Shift + S
- Mac: Command + Shift + 4
3 開発者にSMTP情報を送る
以下の情報を開発者にお送りください: ┌────────────────────────────────────────────── │ │ 1. SMTPサーバー名: sv○○○○.xserver.jp │ │ 2. ポート番号: 465 │ │ 3. 暗号化方式: SSL │ │ 4. メールアドレス: noreply@example.com │ (= SMTPの認証ユーザー名) │ │ 5. パスワード: (設定したパスワード) │ └──────────────────────────────────────────────
パスワードはメールアドレスとは別の方法で送りましょう。
例：サーバー名をメールで送り、パスワードはチャットで送る。
開発者がこの情報を使って、
Webサイトからのメール送信機能を設定できます。
テストメールが届けば設定完了です。
